# 安全政策

## 支持的版本

| 版本 | 支持状态    |
| ---- | ----------- |
| 1.x  | ✅ 活跃支持 |

## 报告安全漏洞

如果您发现安全漏洞，**请不要在公开的 Issue 中报告**。

请通过以下方式私下报告：

1. 发送邮件至项目维护者（邮件地址待补充）
2. 或在 GitHub 上使用 **"Report a vulnerability"** 功能（Security Advisories）

请在报告中包含：

- 漏洞的详细描述
- 重现步骤或概念验证代码
- 受影响的版本
- 可能的修复建议（如有）

## 处理流程

1. 收到报告后，我们将在 **48 小时** 内确认收到
2. 我们将在 **7 天** 内提供初步评估
3. 修复准备好后，我们将发布安全更新
4. 在公开披露前，我们将与报告者协调发布时间

## 最佳实践

使用 MingBo Pro 时，请遵循以下安全建议：

- 在生产环境中修改默认的 `VITE_LOCK_ENCRYPT_KEY`
- 定期更新依赖以获得安全补丁（`pnpm update`）
- 不要在代码中硬编码密钥或敏感信息
- 启用 HTTPS 进行生产部署
